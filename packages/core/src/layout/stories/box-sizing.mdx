# Box Sizing

Utilities for controlling how the browser should calculate an element's total
size.

| Utility      | Properties                 |
| ------------ | -------------------------- |
| `boxBorder`  | `box-sizing: border-box;`  |
| `boxContent` | `box-sizing: content-box;` |

## Include borders and padding

Use `box-border` to set an element's `box-sizing` to `border-box`, telling the
browser to include the element's borders and padding when you give it a height
or width.

This means a 100px × 100px element with a 2px border and 4px of padding on all
sides will be rendered as 100px × 100px, with an internal content area of
88px × 88px.

Tailwind makes this the default for all elements in our preflight base styles

## Exclude borders and padding

Use `box-content` to set an element's `box-sizing` to `content-box`, telling the
browser to add borders and padding on top of the element's specified width or
height.

This means a 100px × 100px element with a 2px border and 4px of padding on all
sides will actually be rendered as 112px × 112px, with an internal content area
of 100px × 100px.

## Responsive

```tsx
<div css={[boxBorder, md(boxContent)]}>{/* ... */}</div>
```
